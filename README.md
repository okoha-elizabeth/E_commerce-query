# E_commerce-query

This repository contains a structured collection of advanced SQL queries built around a customer dataset. The objective of this project is to demonstrate practical SQL skills used in real-world data analysis, moving beyond basic querying into analytical problem-solving.

# Project Overview

The dataset includes customer-level information such as:
1. Customer ID
2. Name
3. City
4. Spend
5. Signup Date

Using this dataset, I developed queries that simulate business-focused analysis scenarios including customer segmentation, revenue analysis, and trend evaluation.

# Key Concepts Covered
1. Data aggregation and grouping
2. Conditional logic using CASE statements
3. Subqueries and nested queries
4. Window functions (RANK, ROW_NUMBER, running totals)
5. Common Table Expressions (CTEs)
6. Business-driven analytical queries

# Sample Analytical Questions
1. Which cities generate the highest total customer spend?
2. Who are the top-performing customers within each region?
3. How does individual customer spend compare to city averages?
4. What are the monthly trends in customer acquisition?
5. How can customers be segmented based on spending behavior?

# Objective

The goal of this project is to strengthen SQL proficiency and demonstrate the ability to transform raw data into structured insights that support decision-making.

# Tools Used
1. SQL (MySQL)
2. MySQL Workbench

# Notes
All queries are written with a focus on clarity, efficiency, and real-world applicability.
